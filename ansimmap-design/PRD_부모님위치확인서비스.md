# PRD: 부모님 위치 확인 서비스

**문서 버전**: v1.0  
**작성일**: 2026-05-24  
**상태**: 초안

---

## 1. 개요 (Overview)

### 1.1 서비스 목적
고령의 부모님 위치를 가족이 실시간으로 확인하고, 긴급 상황 시 신속하게 대응할 수 있도록 지원하는 반응형 웹 기반 가족 안전 서비스.

### 1.2 핵심 가치
- **안심**: 가족이 언제든 부모님의 현재 위치를 파악 가능
- **간편성**: 부모님이 스마트폰 단순 조작만으로 긴급 신호 전송 가능
- **연결성**: 가족 간 채팅·전화 기능으로 즉각적인 소통 지원

### 1.3 구현 방식
- **플랫폼**: 반응형 웹 (PC·태블릿·모바일 모두 지원)
- **주요 화면**: 부모님 화면 / 가족 화면 / 관리자 화면

## 2. 사용자 정의 (User Personas)

| 구분 | 설명 | 주요 니즈 |
|------|------|-----------|
| **부모님** | 디지털 기기 활용에 익숙하지 않은 시니어 | 쉬운 조작, 큰 버튼, 긴급 상황 알림 |
| **가족 구성원** | 스마트폰에 익숙한 자녀·배우자 | 부모님 위치 확인, 빠른 연락, 가족 소통 |
| **서비스 관리자** | 운영사 담당자 | 회원 관리, 이상 징후 모니터링, 고객 커뮤니케이션 |

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 공통 기능

| ID | 기능 | 설명 |
|----|------|------|
| CMN-01 | 회원 가입 / 로그인 | 휴대폰 번호 기반 인증, 가족 그룹 코드 연결 |
| CMN-02 | 가족 그룹 관리 | 부모님 계정과 가족 계정 1:N 연결 |
| CMN-03 | 알림 설정 | 푸시 알림 수신 여부 및 소리/진동 설정 |
| CMN-04 | 위치 정보 동의 | 서비스 이용 전 위치정보 수집 동의 처리 |

### 3.2 부모님 화면 (Parent View)

#### 목표
조작이 단순하고 시인성이 높은 UI로, 고령자도 쉽게 사용할 수 있어야 한다.

| ID | 기능 | 상세 설명 | 우선순위 |
|----|------|-----------|---------|
| PAR-01 | 실시간 위치 공유 | 앱 실행 시 GPS 위치를 가족에게 자동 공유. 백그라운드 위치 전송 지원 | P0 |
| PAR-02 | 긴급 알림 버튼 | 화면 중앙에 크게 배치된 SOS 버튼. 한 번 탭하면 연결된 모든 가족에게 즉시 푸시 알림 전송 | P0 |
| PAR-03 | 버튼 오조작 방지 | 긴급 버튼 탭 후 2초 카운트다운 또는 "길게 누르기" 방식으로 오발송 방지 | P1 |
| PAR-04 | 긴급 알림 내용 | 알림에 현재 위치(주소 + 지도 링크) 및 발송 시각 포함 | P0 |
| PAR-05 | 배터리 절약 모드 | 위치 전송 주기를 조절하여 배터리 소모 최소화 | P2 |
| PAR-06 | 상태 메시지 | "정상", "외출 중" 등 간단한 상태를 가족에게 전달하는 기능 | P2 |

#### UI 지침
- 글자 크기: 최소 18px 이상
- 버튼 크기: 최소 48×48px 이상 (긴급 버튼은 화면의 1/3 이상)
- 색상 대비: WCAG AA 기준 충족
- 화면 내 최소한의 UI 요소만 배치 (정보 과부하 금지)

### 3.3 가족 화면 (Family View)

#### 목표
부모님의 현재 위치를 직관적으로 확인하고, 빠른 연락 및 가족 소통이 가능해야 한다.

| ID | 기능 | 상세 설명 | 우선순위 |
|----|------|-----------|---------|
| FAM-01 | 실시간 위치 지도 | 지도(카카오맵 또는 네이버맵 API) 위에 부모님 현재 위치 마커 표시. 주소 텍스트 함께 표시 | P0 |
| FAM-02 | 위치 이력 조회 | 당일 이동 경로를 지도에서 확인 가능 (최대 7일 이력) | P1 |
| FAM-03 | 바로 전화 연결 버튼 | 버튼 한 번으로 부모님 휴대폰에 바로 전화 연결 (tel: 링크 방식) | P0 |
| FAM-04 | 가족 채팅방 | 가족 그룹 내 단체 채팅. 텍스트·이미지 전송 지원 | P0 |
| FAM-05 | 긴급 알림 수신 | 부모님이 SOS 버튼 누를 시 푸시 알림 + 인앱 팝업으로 즉시 안내 | P0 |
| FAM-06 | 마지막 확인 시각 | 부모님 위치가 마지막으로 업데이트된 시각 표시 | P1 |
| FAM-07 | 다수 부모님 지원 | 연결된 부모님이 여러 명일 경우 리스트 형태로 각각 위치 확인 가능 | P2 |

### 3.4 관리자 화면 (Admin View)

#### 목표
서비스 이용 회원(가족 그룹) 관리 및 고객 커뮤니케이션을 효율적으로 처리한다.

| ID | 기능 | 상세 설명 | 우선순위 |
|----|------|-----------|---------|
| ADM-01 | 회원(가족) 목록 조회 | 가입된 가족 그룹 목록. 그룹명, 멤버 수, 가입일, 이용 현황 표시 | P0 |
| ADM-02 | 회원 상세 정보 | 그룹 내 개별 회원 정보(이름, 연락처, 역할, 마지막 접속일) 확인 | P0 |
| ADM-03 | 회원 검색 및 필터 | 이름·전화번호로 검색, 상태(활성/비활성)별 필터 | P1 |
| ADM-04 | 문자(SMS) 발송 | 특정 회원 또는 전체 회원에게 문자 메시지 직접 발송 | P0 |
| ADM-05 | 카카오톡 알림 발송 | 카카오 알림톡 API 연동을 통해 카카오톡 메시지 발송 | P0 |
| ADM-06 | 발송 이력 조회 | SMS/카카오톡 발송 이력 및 수신 상태 확인 | P1 |
| ADM-07 | 서비스 이용 통계 | 일/주/월별 활성 사용자, 긴급 알림 발생 건수 등 통계 대시보드 | P2 |
| ADM-08 | 계정 정지/복구 | 비정상 이용 회원 계정 정지 및 복구 처리 | P1 |

## 4. 비기능 요구사항 (Non-Functional Requirements)

| 구분 | 요구사항 |
|------|---------|
| **성능** | 위치 업데이트 지연 시간 5초 이내, 긴급 알림 전송 3초 이내 |
| **가용성** | 서비스 uptime 99.5% 이상 |
| **보안** | 위치 데이터 암호화 전송(HTTPS/TLS), 개인정보보호법 준수 |
| **반응형** | 모바일(360px~), 태블릿(768px~), PC(1280px~) 모두 지원 |
| **접근성** | 시니어 사용자를 위한 큰 텍스트·버튼, 화면 확대 지원 |
| **브라우저** | Chrome, Safari, Samsung Internet 최신 2버전 이상 지원 |

## 5. 기술 스택 제안

| 영역 | 기술 |
|------|------|
| 프론트엔드 | React.js (반응형, PWA 지원) |
| 백엔드 | Node.js / Express 또는 Spring Boot |
| 데이터베이스 | PostgreSQL (회원·채팅 데이터) + Redis (실시간 위치 캐싱) |
| 지도 API | 카카오맵 API 또는 네이버 지도 API |
| 실시간 통신 | WebSocket (Socket.io) — 실시간 위치 + 채팅 |
| 푸시 알림 | FCM (Firebase Cloud Messaging) |
| 카카오톡 | 카카오 알림톡 API |
| SMS | NHN Cloud Notification / 네이버 클라우드 SMS |

## 6. 화면 구성 요약

```
서비스 화면 구조
├── 공통
│   ├── 로그인 / 회원가입
│   └── 가족 그룹 코드 연결
│
├── 부모님 화면
│   ├── 메인 (SOS 버튼 + 위치 공유 상태)
│   └── 설정 (알림·위치 권한)
│
├── 가족 화면
│   ├── 위치 지도 (실시간)
│   ├── 위치 이력
│   ├── 전화 연결 버튼
│   └── 가족 채팅방
│
└── 관리자 화면
    ├── 대시보드 (통계)
    ├── 회원 관리 (목록·상세)
    ├── 메시지 발송 (SMS·카카오톡)
    └── 발송 이력
```

## 8. 주요 고려사항 및 리스크

| 항목 | 내용 |
|------|------|
| **위치정보 법규** | 위치정보의 보호 및 이용 등에 관한 법률 준수 필요, 이용자 동의 절차 필수 |
| **배터리 소모** | 백그라운드 위치 추적으로 인한 부모님 기기 배터리 소모 최소화 전략 필요 |
| **시니어 UX** | 부모님 화면은 실제 시니어 사용자 대상 UX 테스트 권장 |
| **긴급 오발신** | SOS 오발신으로 인한 가족 혼란 방지를 위한 오조작 방지 로직 필수 |
| **카카오 API 비용** | 카카오 알림톡은 건당 과금이므로 발송 정책 수립 필요 |
