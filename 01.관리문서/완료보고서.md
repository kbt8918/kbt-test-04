## 6. 교훈 (Lessons Learned)

### 6.1 잘된 점

- 프로토타입(PrototypeShell) 기반 화면 구조를 선행 확보해 P0 3개 화면을 빠르게 구현했습니다.
- 화면설계서(SCR-001~014)와 구현 코드의 화면 ID를 일치시켜 추적성을 확보했습니다.
- 구글 로그인 인증 우회 취약점·민감 정보 추적 문제를 개발 단계에서 선제 발견·수정했습니다.

### 6.2 개선할 점

- 테스트 코드·CI를 구현과 병행하지 못해 W5로 이월되었습니다. 향후 기능 구현과 테스트 작성을 동시 진행할 필요가 있습니다.
- 실시간 통신을 폴링으로 단순화했으므로, 트래픽 확장 시 WebSocket 전환 설계를 사전 준비해야 합니다.
- 외부 API(FCM·SMS·알림톡) 실연동 검증을 개발 후반으로 미뤄 검수 부담이 집중되었습니다.
