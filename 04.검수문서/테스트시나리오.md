## n8n 자동 테스트 매트릭스

본 절은 위 시나리오 중 **HTTP로 검증 가능하고 데이터를 변경하지 않는** 케이스를, n8n 워크플로우가 **읽어서 그대로 실행**할 수 있도록 기계가 파싱 가능한 형태로 정리한 것이다. n8n은 아래 `N8N-TEST-MATRIX` 구간의 표를 파싱하여 케이스별로 호출하고 Pass/Fail을 판정한다.

- `method`: HTTP 메서드 (GET/POST)
- `path`: 요청 경로 (BASE 주소 뒤에 붙음)
- `body`: POST 본문(JSON). 없으면 `-`
- `expectStatus`: 기대 HTTP 상태코드
- `keyword`: 응답 본문에 포함되어야 할 문자열. 없으면 `-`

> 새 케이스를 추가하려면 아래 표에 한 줄을 더하면 된다. 워크플로우 코드 수정 없이 시나리오만 늘릴 수 있다. (서술형 TC를 실행 가능한 한 줄로 옮기는 것은 PM의 "무엇을 검증할지" 설계 작업이다.)

<!-- N8N-TEST-MATRIX:START -->
| TC | name | method | path | body | expectStatus | keyword |
