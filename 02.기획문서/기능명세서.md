### F-001. 회원가입

| 항목 | 내용 |
|------|------|
| 설명 | 휴대폰 번호와 역할(부모님 / 가족 구성원) 선택만으로 3단계 이내에 회원가입을 완료한다. 앱 설치 없이 모바일 웹 브라우저에서 처리된다. |
| 입력 | 휴대폰 번호 (11자리 숫자), 역할 선택 (parent \| family), 서비스 이용약관 동의 체크 |
| 처리 | 1. 휴대폰 번호 형식 검증 (010-XXXX-XXXX)<br>2. 중복 번호 여부 DB 조회<br>3. 역할 및 동의 여부 저장<br>4. users 테이블에 신규 레코드 삽입<br>5. JWT 액세스 토큰(7일) 발급 및 httpOnly 쿠키 저장 |
| 출력 | 가입 완료 화면 전환 + JWT 세션 쿠키 저장<br>역할에 따라 온보딩 플로우(F-005) 또는 가족 화면으로 분기 |
| 예외 처리 | 중복 번호: "이미 가입된 휴대폰 번호입니다. 로그인을 진행해주세요." 에러 필드 노출<br>구글 연동 중복: "이미 연동된 구글 계정입니다. 로그인 탭을 이용해주세요." 상단 배너<br>형식 오류/약관 미동의: 입력 필드 하단 인라인 에러 및 상단 배너 표시 |
| 관련 화면 | 회원가입 화면 (Parent View / Family View 공통) |
| 관련 API | `POST /api/auth/register` |

### F-002. 로그인

| 항목 | 내용 |
|------|------|
| 설명 | 등록된 휴대폰 번호 입력으로 로그인하고, 7일간 자동 로그인 세션을 유지한다. |
| 입력 | 휴대폰 번호 (11자리 숫자) |
| 처리 | 1. 번호 형식 검증<br>2. users 테이블에서 번호 조회<br>3. 일치 시 JWT 액세스 토큰 재발급 및 쿠키 갱신<br>4. 역할(role)과 그룹 가입 여부, 부모님 유무에 따른 스마트 라우팅 분기 |
| 출력 | 역할에 따라 부모님 화면(Parent View) 또는 가족 화면(Family View), 미설정 시 가족 설정(family-settings)으로 이동 |
| 예외 처리 | 미가입 번호: "등록되지 않은 번호입니다. 회원가입을 진행해주세요." 에러 필드 노출<br>형식 오류: 인라인 에러 메시지<br>서버 에러: 상단 에러 배너 및 "잠시 후 다시 시도해 주세요." 토스트 |
| 관련 화면 | 로그인 화면 (공통) |
| 관련 API | `POST /api/auth/login`, `POST /api/auth/logout`, `GET /api/auth/me` |

### F-003. 가족 그룹 생성

| 항목 | 내용 |
|------|------|
| 설명 | 가족 구성원(자녀)이 가족 그룹을 생성하면 6자리 초대 코드와 QR 코드가 자동 발급된다. |
| 입력 | 그룹명 (최대 20자), 생성자 user_id |
| 처리 | 1. 그룹명 공백·길이 검증<br>2. 6자리 영숫자 초대 코드 랜덤 생성 (중복 체크 포함)<br>3. groups 테이블에 그룹 레코드 삽입<br>4. group_members 테이블에 생성자를 owner 역할로 삽입<br>5. 초대 코드 기반 QR 코드 생성 (qrcode 라이브러리) |
| 출력 | 그룹 상세 화면 이동 + 초대 코드 및 QR 코드 표시 |
| 예외 처리 | 그룹명 미입력: 생성 버튼 비활성화<br>초대 코드 중복 충돌: 자동 재생성 (최대 3회 재시도 후 서버 에러 반환) |
| 관련 화면 | 그룹 생성 화면 (Family View) |
| 관련 API | `POST /api/groups` |

### F-004. 가족 그룹 참여

| 항목 | 내용 |
|------|------|
| 설명 | 초대 코드 또는 QR 코드 스캔으로 기존 가족 그룹에 참여한다. Free 플랜 기준 그룹당 최대 5명까지 참여 가능하다. |
| 입력 | 초대 코드 (6자리) 직접 입력 또는 `/join?code=` 링크 클릭 후 진입 |
| 처리 | 1. `/join?code=` 링크로 진입 시 상태 저장, 회원가입/로그인 후 자동 그룹 참여<br>2. 초대 코드 형식 검증<br>3. groups 테이블에서 코드 일치 그룹 조회<br>4. 정원 및 중복 참여 확인<br>5. group_members 테이블에 member 역할로 삽입 |
| 출력 | 가족 화면(Family View) 진입 또는 가족 설정 완료 화면 이동 |
| 예외 처리 | 유효하지 않은 코드: "초대 코드를 다시 확인해 주세요." 에러 배너 및 토스트<br>그룹 미가입/데모 시 링크 복사 시도: "가족 그룹을 먼저 생성해주세요" 안내<br>정원 초과: 안내 메시지 노출 |
| 관련 화면 | 코드 입력 화면 (Family View) |
| 관련 API | `POST /api/groups/join`, `GET /api/groups/:id` |

### F-005. 온보딩 플로우 (자녀 대리 설정)

| 항목 | 내용 |
|------|------|
| 설명 | 자녀가 부모님 단말기를 대리하여 회원가입·그룹 참여·위치정보 동의까지 단계별로 안내하는 온보딩 흐름을 제공한다. 마지막 단계에서 부모님 본인의 명시적 동의 화면으로 전환된다. |
| 입력 | 각 단계별 사용자 입력 (번호, 역할 선택, 코드 입력, 동의 체크) |
| 처리 | 1단계: 부모님 번호 입력 및 역할 선택 (parent 고정)<br>2단계: 그룹 초대 코드 입력 (F-004 실행)<br>3단계: 위치정보 동의 화면 전환 (부모님 본인 조작 필수 안내)<br>4단계: 동의 완료 후 부모님 화면(Parent View) 진입 |
| 출력 | 부모님 화면 메인 (SOS 버튼 중심 화면) 진입 |
| 예외 처리 | 중간 단계 이탈: 진행 상태 localStorage에 임시 저장, 재접속 시 이어서 진행<br>동의 거부: 위치 공유 비활성 상태로 부모님 화면 진입, 동의 유도 배너 표시 |
| 관련 화면 | 온보딩 1~4단계 화면 (Parent View) |
| 관련 API | F-001, F-004, F-006 API 순차 호출 |

### F-006. 위치정보 수집 동의

| 항목 | 내용 |
|------|------|
| 설명 | 위치정보법 제15조에 따라 부모님 단말기 화면에서 위치정보 수집·이용·제공에 대한 명시적 사전 동의를 받는다. 자녀의 대리 조작으로 강제 동의 처리는 불가하다. |
| 입력 | 동의 항목 체크 (필수: 위치정보 수집·이용 동의 / 선택: 마케팅 활용 동의), 부모님 본인 확인 (휴대폰 번호 재확인) |
| 처리 | 1. 동의 화면을 부모님 단말기 기준 전체 화면으로 표시<br>2. 필수 동의 체크 여부 확인<br>3. location_consents 테이블에 동의 일시·IP·버전 저장<br>4. users 테이블의 location_consent_at 필드 갱신<br>5. 브라우저 Geolocation API 권한 요청 프롬프트 표시 |
| 출력 | 동의 완료 상태 저장 + 위치 공유 활성화 + 부모님 메인 화면 진입 |
| 예외 처리 | 브라우저 위치 권한 거부: "위치 권한을 허용해야 서비스를 이용할 수 있습니다." 안내 + 설정 방법 가이드 링크<br>필수 항목 미동의: 확인 버튼 비활성화 |
| 관련 화면 | 위치정보 동의 화면 (Parent View — 전체 화면 단독 표시) |
| 관련 API | `POST /api/location/consent` |

### F-008. 위치정보 동의 철회

| 항목 | 내용 |
|------|------|
| 설명 | 부모님이 설정 화면에서 위치정보 제공 동의를 언제든지 철회할 수 있다. 철회 즉시 위치 수집이 중단되고 가족에게 알림이 발송된다. |
| 입력 | 동의 철회 버튼 탭 + 확인 다이얼로그 승인 |
| 처리 | 1. 확인 다이얼로그로 이중 확인<br>2. users 테이블의 location_consent_at을 null로 갱신<br>3. location_consents 테이블에 철회 이력 저장<br>4. 위치 전송 스케줄러 즉시 중단<br>5. 가족 구성원 전원에게 FCM 푸시 알림 발송: "부모님께서 위치 공유를 중단했습니다." |
| 출력 | 위치 공유 OFF 상태 전환 + 가족 화면 지도에서 부모님 마커 제거 |
| 예외 처리 | 서버 에러: 철회 실패 토스트 메시지, 위치 수집 상태 유지 |
| 관련 화면 | 설정 화면 (Parent View) |
| 관련 API | `DELETE /api/location/consent` |

### F-009. 실시간 GPS 위치 전송

| 항목 | 내용 |
|------|------|
| 설명 | 부모님 단말기의 GPS 위치를 30초 간격으로 서버에 전송한다. 브라우저 Geolocation API를 사용하며, 위치 공유 동의 상태이고 ON 토글일 때만 동작한다. |
| 입력 | 브라우저 Geolocation API의 위도(latitude), 경도(longitude), 정확도(accuracy), 타임스탬프 |
| 처리 | 1. 위치정보 동의 상태 및 ON 토글 여부 확인<br>2. `navigator.geolocation.watchPosition()` 등록 (30초 간격)<br>3. 위치 데이터를 `POST /api/location` 엔드포인트로 전송<br>4. locations 테이블에 위도·경도·타임스탬프 저장<br>5. WebSocket을 통해 동일 그룹의 가족 화면에 실시간 전파<br>6. 7일 초과 위치 이력 자동 삭제 스케줄러 실행 |
| 출력 | locations 테이블 업데이트 + 가족 화면 지도 마커 실시간 갱신 |
| 예외 처리 | Geolocation 권한 거부: 위치 공유 OFF 상태로 전환 + 권한 허용 안내<br>GPS 신호 불량(accuracy > 100m): 마지막 정상 위치 유지, 불량 데이터 저장 제외<br>네트워크 단절: 로컬 큐에 최대 5건 임시 저장 후 재연결 시 일괄 전송<br>iOS Safari 백그라운드 전환: 포그라운드 유지 안내 배너 표시 |
| 관련 화면 | 부모님 메인 화면 (Parent View — 백그라운드 동작) |
| 관련 API | `POST /api/location`, WebSocket `/ws` |

### F-029. 다중 부모님 동시 위치 확인

| 항목 | 내용 |
|------|------|
| 설명 | 가족이 한 그룹에 등록된 여러 부모님(예: 아버지·어머니)의 최신 위치를 한 화면에서 동시에 확인한다. 요금제(플랜)별 동시 확인 한도를 적용하며, 한도 초과 부모님은 위치를 가리고 업그레이드를 유도한다. |
| 입력 | group_id (현재 가족 그룹), 요청자 JWT |
| 처리 | 1. 요청자의 그룹 구성원 여부 확인<br>2. 그룹 plan 조회 (Free 2명 / Pro 무제한)<br>3. 그룹 내 부모님(role=parent) 목록 조회 후 plan 한도까지만 위치 노출<br>4. 각 부모님의 위치 공유(location_sharing) ON 상태에 한해 locations 테이블 최신 1건 조회<br>5. 부모님별 이름·호칭(relation)·좌표·측정 시각·공유 상태를 묶어 반환 |
| 출력 | 부모님별 위치 카드/마커 목록 + 플랜·한도(planLimit)·전체 부모 수(totalParents)·노출 수(visibleCount) |
| 예외 처리 | 한도 초과 부모님: 위치 비노출 + "Pro 플랜에서 모든 부모님 위치를 확인할 수 있습니다." 업그레이드 유도<br>위치 공유 OFF: 좌표 null 반환 및 "위치 공유 중지" 표시<br>비구성원 접근: 403 반환 |
| 관련 화면 | 가족 메인 화면 (Family View, SCR-006) |
| 관련 API | `GET /api/location/current/:groupId/parents` |

### F-030. 부모님 호칭(관계) 지정

| 항목 | 내용 |
|------|------|
| 설명 | 가족이 그룹 내 부모님 구성원에게 호칭(어머니·아버지 등)을 지정하거나 해제한다. 지정된 호칭은 지도 마커·위치 카드·채팅 등 가족 화면 전반에 표시된다. |
| 입력 | group_id, 대상 userId, relation (호칭 문자열 1~20자, 빈 값/null이면 해제) |
| 처리 | 1. 요청자와 대상이 모두 동일 그룹 구성원인지 확인<br>2. 호칭 정규화: 빈 값이면 null(해제), 값이 있으면 1~20자 형식 검증<br>3. group_members 테이블의 relation 컬럼 갱신<br>4. 갱신된 userId·relation 반환 |
| 출력 | 대상 구성원의 호칭 표시 갱신 (가족 화면 마커·카드) |
| 예외 처리 | 대상 userId 누락: "대상 사용자가 필요합니다." 반환<br>호칭 형식 오류(1~20자 초과): "호칭은 1~20자여야 합니다." 반환<br>비구성원 대상/요청자: 403 반환 |
| 관련 화면 | 가족 메인 화면·가족 설정 화면 (Family View, SCR-006/010) |
| 관련 API | `PATCH /api/groups/:id/members` |
